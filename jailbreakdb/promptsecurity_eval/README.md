# PromptSecurity-Eval

This subtree extends JailbreakDB with evaluation-result records generated by the
PromptSecurity evaluation protocol. It intentionally uses a schema separate
from prompt-corpus records: prompt corpora store prompt text, while
PromptSecurity-Eval stores CSV measurements over configured `<model, attack,
defense, dataset, judger>` tuples.

## Contents

- `data/sample_records/harmful_main_parts/part-*.csv.gz`: sharded main harmful-query sample-level records.
- `data/sample_records/benign_utility.csv.gz`: benign utility sample-level records.
- `data/sample_records/baseline_calibration.csv.gz`: baseline calibration sample-level records.
- `data/run_records/*.csv.gz`: run-level metadata records.
- `data/prompt_samples/balanced_challenge.csv.gz`: the selected 100-prompt harmful-query sample.
- `schema.json`: compact schema description.
- `manifest.json`: row counts and source split metadata.

## Splits

| Split | Run files | Sample records | Completed sample records | Records with responses |
|---|---:|---:|---:|---:|
| `harmful_main` | 1,672 | 167,200 | 161,849 | 158,287 |
| `benign_utility` | 92 | 9,200 | 9,200 | 9,195 |
| `baseline_calibration` | 87 | 8,700 | 8,700 | 8,557 |

## Split Semantics

- `harmful_main`: main harmful-query evaluation over attack and defense configurations.
- `benign_utility`: normal-question utility evaluation for defense behavior on benign inputs.
- `baseline_calibration`: calibration runs used to select and audit the fixed harmful-query subset.

## Loading

```python
from pathlib import Path

import pandas as pd

base = Path("jailbreakdb/promptsecurity_eval/data")

sample_records = {
    "harmful_main": pd.concat(
        [
            pd.read_csv(path)
            for path in sorted((base / "sample_records/harmful_main_parts").glob("part-*.csv.gz"))
        ],
        ignore_index=True,
    ),
    "benign_utility": pd.read_csv(base / "sample_records/benign_utility.csv.gz"),
    "baseline_calibration": pd.read_csv(base / "sample_records/baseline_calibration.csv.gz"),
}

run_records = {
    "harmful_main": pd.read_csv(base / "run_records/harmful_main.csv.gz"),
    "benign_utility": pd.read_csv(base / "run_records/benign_utility.csv.gz"),
    "baseline_calibration": pd.read_csv(base / "run_records/baseline_calibration.csv.gz"),
}

prompt_samples = pd.read_csv(base / "prompt_samples/balanced_challenge.csv.gz")
```

## Important Fields

Sample-level CSV files include prompt identifiers, clean and attacked prompts,
model responses, model/attack/defense identifiers, execution status, method
configuration JSON, query-cost fields, and normalized independent judger outputs:

- `judger_gpt_harmful_binary`
- `judger_harmbench`
- `judger_rejection_prefix`
- `judger_utility`

Run-level CSV files summarize each raw experiment run, including configured
model, attack, defense, dataset, judgers, sample counts, completion status, and
run metadata.
